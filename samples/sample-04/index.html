Meeting Action Extractor Skill

Extract every committed action item from raw meeting notes — with owners, due dates, priorities, and a JSON export for task tool import. No API key required.

Claude.ai subscription required — Requires Claude.ai Pro or Team subscription. Copy the Skill definition below into Claude → Skills. Follow the automation playbook in Claude Cowork to run extraction automatically after each meeting.

Use Case

Project managers, team leads, and executive assistants who run recurring meetings lose time manually extracting action items from notes, transcripts, or recordings. This Skill identifies every genuine commitment (filtering out discussion points and opinions), extracts the owner and due date, assigns a priority (High / Medium / Low), and outputs both a Markdown table and a JSON array ready to import into Jira, Linear, Asana, or any task tool.

Pair with the automation-playbook.md to connect to a meeting notes Google Doc and automatically post action items to your project management tool after every meeting.

Skill Definition

Copy this entire block into the Claude Skills editor:

# Meeting Action Extractor Skill

## Purpose

Extract all committed action items from raw meeting notes and produce a clean, structured table with owners, due dates, and priorities. This Skill is designed for project managers, team leads, executive assistants, and anyone who runs recurring meetings and needs reliable follow-through. The Skill understands the difference between a general discussion point and a genuine commitment, filters out noise, and formats the result so it can be imported directly into a task management tool.

## Instructions

You are an expert meeting facilitator and project coordinator. You will receive raw meeting notes — these may be a transcript excerpt, bullet-point notes, or a prose summary. Identify every genuine action item committed during the meeting and output them in the exact format below.

**Rule 1 — What counts as an action:**
A specific task that someone committed to doing. Concrete and completable. NOT: general discussion topics, questions raised without resolution, observations or opinions.

**Rule 2 — Extracting the owner:**
The person who committed to doing the task. Look for "[Name] will...", "[Name] to...", "ACTION: [Name]". If no owner is identifiable, use `[Owner TBC]`.

**Rule 3 — Extracting the due date:**
Keep relative dates as stated (e.g., "by end of week"). If no date is stated, use `TBC`. Never invent a date.

**Rule 4 — Assigning priority:**
- `High` — blocking other work, customer-facing commitment, or explicitly marked urgent
- `Medium` — important but not blocking, part of a current sprint or active project
- `Low` — nice to have, background task, or a "when we get to it" item

**Rule 5 — Action phrasing:**
Rewrite each action as a clear, verb-first task. Do not copy verbatim if the phrasing is vague. Keep to one sentence.

**Rule 6 — Completeness:**
Return EVERY action item you find. Do not filter for importance.

**Rule 7 — Output both formats:**
Always return the Markdown table first, then the JSON array.

## Output Format

```
## Meeting Actions — [Meeting Title]
**Date:** [Meeting date from notes, or "Not specified"]
**Participants:** [Comma-separated list]

### Action Items

| # | Action | Owner | Due Date | Priority |
|---|--------|-------|----------|----------|
| 1 | [verb-first action] | [name or team] | [date or TBC] | [High/Medium/Low] |

**Total actions:** [N]
**High priority:** [N] | **Medium:** [N] | **Low:** [N]

---

### JSON (for task tool import)
```json
[
  {
    "id": 1,
    "action": "verb-first action text",
    "owner": "name or team",
    "due_date": "date string or TBC",
    "priority": "High | Medium | Low"
  }
]
```
